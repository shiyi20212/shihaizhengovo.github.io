CSS背景图片完全显示的几种方法

发布时间：2026-01-06 分类：CSS技巧 阅读量：96

在网页开发中，背景图片经常需要完整显示且不拉伸变形，本文总结了contain、cover等属性的用法，以及不同场景下的适配方案，解决背景图片裁剪、拉伸的问题...

一、背景图显示不全的原因

网页中背景图默认会“平铺”（重复显示），且不会自动适配容器尺寸——如果图片宽高比和容器不一致，要么被裁剪，要么被拉伸变形。 解决这个问题，核心是用CSS的background-size属性控制图片的显示方式。

二、方法1：contain（完整显示，留空白）

contain会让背景图“等比例缩放”，直到图片的宽度/高度贴合容器边界，保证图片完整显示，但容器可能出现空白区域（因为图片比例和容器不一致）。

/* HTML：给容器加类名 */ <div class="bg-box contain"></div> /* CSS样式 */ .bg-box { width: 500px; height: 300px; background-image: url("bg.jpg"); background-repeat: no-repeat; /* 禁止平铺 */ background-position: center; /* 图片居中 */ } .contain { background-size: contain; /* 完整显示图片 */ }

适用场景：需要完整展示图片内容（比如logo、海报），允许容器留空白。

三、方法2：cover（填满容器，裁边缘）

cover会让背景图“等比例缩放”，直到图片填满整个容器，没有空白区域，但图片超出容器的部分会被裁剪。

/* CSS样式 */ .cover { background-size: cover; /* 填满容器，裁剪边缘 */ }

适用场景：背景图是装饰性图片（比如banner图），不需要完整显示细节，优先填满容器。

四、方法对比与场景选择

属性值 效果 适用场景
contain 图片完整显示，容器可能留空白 需要展示完整图片内容（如logo、信息图）
cover 填满容器，图片可能被裁剪 装饰性背景图（如banner、卡片背景）

实际开发中，还可以结合background-position调整裁剪区域（比如background-position: top让图片顶部对齐，裁剪底部），让背景图显示更符合需求。
